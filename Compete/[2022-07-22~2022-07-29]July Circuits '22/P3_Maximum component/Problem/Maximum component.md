### Maximum component

<p>You are given a graph consisting of <em>N</em> nodes. Initially, there is no edge in the graph. You have to process <em>Q</em> queries of the following two types:</p>
<ul>
	<li><em>1 X Y:</em> connect nodes <em>X</em> and <em>Y</em> with an undirected edge.</li>
	<li><em>2 K:</em> Print the size of the largest connected component if <em>K</em> extra undirected edges are added between any pair of nodes of the graph.</li>
</ul>
<p>Note that the edges connected in the type <em>2</em> query are not considered in further queries.</p>
<p><strong>Note: </strong>Since the size of input-output is large, prefer using fast input-output methods.</p>
<p><strong>Input format</strong></p>
<ul>
	<li>The first line of input contains <em>N</em> denoting the number of nodes and <em>Q</em> denoting the number of queries respectively.</li>
	<li><em>Q</em> lines follow. Each of these lines contains an integer<em> type[i],</em> denoting the type of the<em> i-</em>th query. if <em>type[i] </em>is <em>1</em>, it contains two more integers <em>X, Y</em>,  if <em>type[i] = 2,</em> it contains an integer <em>K</em>.</li>
</ul>
<p><strong>Output format</strong><br>For each query of type 2, print the size of the largest connected component in a separate line.</p>
